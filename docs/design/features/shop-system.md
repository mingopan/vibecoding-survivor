# 商店系统

## 概述

每回合战斗结束后进入**商店界面**。玩家消耗**代币**购买武器、武器扩展或刷新商品栏。商店是局内构筑的核心决策点。

## 商店流程

```
进入商店（显示当前代币、武器槽、各武器扩展槽状态）
  → 浏览商品 / 刷新
  → 购买（扣代币、物品进入对应槽位）
  → 可选：合成（见武器系统）
  → 点击「开始下一回合」
```

## 商品类型

| 类型 | 说明 | 去向 |
|------|------|------|
| 武器 | 新武器本体 | 空武器槽；槽满则不可购买 |
| 武器扩展 | 绑定单把武器的模块 | 指定武器的扩展槽 |
| 全局扩展 | 全局 Buff，不绑定具体武器 | 特殊全局槽或「任意武器」扩展规则 |
| 消耗品（可选） | 下回合一次性效果 | 若实现，占用扩展槽或即时使用 |

### 武器扩展子类

- **数值扩展**：攻速、伤害、弹道数量等。
- **行为扩展**：改变发射方式、子弹形态、命中效果。
- **全局 Buff 扩展**：全队/全武器生效（如移速、拾取范围），仍占用某武器的扩展槽或独立全局槽位（实现时二选一，建议独立 1~2 个「全局扩展槽」避免挤占武器 BD 槽）。

## 商品栏

| 参数 | 建议默认值 | 说明 |
|------|------------|------|
| 固定栏位数 | 4~6 | 每回合进入商店时生成 |
| 刷新费用 | 递增或固定 | 如首次免费，之后 +N 代币 |
| 品质权重 | 随波次提高 | 后期高品阶出现率上升 |

每件商品展示：**底图品质色** + 名称 + 简短描述 + 价格。

## 购买规则

1. **代币不足** → 购买失败，提示。
2. **武器槽已满**（6/6）→ 不可再买武器，除非先合成或「出售/替换」机制（首版可不做出售，仅合成腾槽）。
3. **扩展槽已满** → 需选择要替换的扩展，或购买失败。
4. 购买成功后立即生效，下回合战斗可用。

## 合成（商店内操作）

当玩家拥有**两把相同 `weapon_type` 且相同 `quality`** 的武器时，可在商店触发合成：

```
武器 A（品质 Q）+ 武器 B（品质 Q）→ 武器 A'（品质 Q，数值提升）
```

- 合成后**仅占 1 个武器槽**，释放一个槽位。
- 扩展槽内容：建议**合并**（同 ID 扩展叠加层数）或**保留主武器**上的扩展（实现细则见 [weapon-system.md](weapon-system.md)）。
- 合成不消耗代币，但消耗一把副武器。

## UI 要点

- 左侧：当前装备（6 武器槽 + 每把武器的扩展槽一览）。
- 中央：商品列表。
- 右侧：选中商品详情、购买/刷新按钮。
- 底部：代币数量、「下一回合」按钮（槽位未满时也可继续，不强制买满）。
