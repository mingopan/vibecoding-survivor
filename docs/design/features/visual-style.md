# 视觉风格

## 方向

**2D 极简多边形 + 辉光（Glow）+ Shader 后处理**，在低成本美术下追求辨识度与打击感。与幸存者类大量同屏单位的需求兼容。

## 画面分层

```
后处理层（全屏 shader：辉光、色差、暗角、轻微颗粒）
  ↑
VFX 层（命中闪、弹道尾迹、升级光晕）
  ↑
实体层（玩家、怪物、弹体 — 极简多边形）
  ↑
地面层（简单网格或纯色渐变 + 微弱扫描线）
```

## 极简多边形

### 造型原则

- 所有角色、怪物、弹体用 **少顶点 polygon**（三角、四边、六边）构成，避免手绘_sprite 依赖。
- 外形靠 **轮廓 + 颜色 + 动画** 区分，不靠细节纹理。
- 玩家与敌人色相分离：玩家冷色（青/蓝），敌人暖色（红/橙）或高饱和紫。

### 示例映射

| 对象 | 形状 | 备注 |
|------|------|------|
| 玩家 | 六边形 + 方向缺口 | 缺口指示面朝 |
| 小型怪 | 三角形 | 成群时易读 |
| 中型怪 | 菱形 / 五边形 | 体积略大 |
| Boss | 组合多边形 + 外圈环 | 额外描边 |
| 子弹 | 细长四边形 / 三角 | 朝向速度方向 |
| 范围技 | 圆环 polygon 近似 | 半透明填充 |

### 动画

- 移动：轻微 squash & stretch。
- 受击：闪白 + 顶点抖动。
- 死亡：碎裂为小三角淡出（粒子可用同色 polygon）。

## 辉光（Glow）

### 用途

- 品质 ≥ 史诗 的道具 UI 与掉落物。
- 枪口焰、命中火花、技能充能。
- 玩家武器开火时的短暂轮廓光。

### 实现思路（Godot 4）

1. **物体级**：`CanvasItem` 材质 + `shader` 外扩采样 / 加法混合。
2. **屏幕级**：`Glow` 后处理（见下节）对亮部区域模糊叠加。

### 参数与品质联动

从 [item-quality.md](item-quality.md) 读取 `glow_strength`：

| 品质 | 辉光强度 | 脉冲 |
|------|----------|------|
| 1~3 | 0 | 无 |
| 4 | 0.3 | 慢速 |
| 5 | 0.6 | 中速 |
| 6 | 1.0 | 快速 + 色相轻微漂移 |

## Shader 后处理

建议挂载在 `CanvasLayer` 或 `SubViewport` 全屏 `ColorRect` 上，统一调色。

| 效果 | 强度建议 | 目的 |
|------|----------|------|
| Bloom / Glow | 中 | 强化亮点与品质感 |
| 暗角 Vignette | 弱 | 聚焦中心战斗区 |
| 色差 Chromatic Aberration | 极弱 | 命中/Boss 出现时瞬时加强 |
| 颗粒 Film Grain | 极弱 | 避免纯色太平 |
| 色调映射 | 略提对比 | 多边形边缘更清晰 |

### 性能

- 移动端可关色差与颗粒，保留轻量 bloom。
- 同屏怪物 >80 时降低个体辉光，依赖屏幕级 bloom。

## UI 与品质色

商店、HUD 道具框使用品质底图色（与 item-quality 表一致），图标仍为极简线框 polygon。

- 字体：无衬线，粗体标题 + 常规正文。
- 代币、倒计时：大字号固定角落，高对比。

## 战斗可读性

| 原则 | 做法 |
|------|------|
| 弹道清晰 | 高对比色 + 尾迹；玩家弹偏亮，敌人弹偏暗红 |
| 伤害数字 | 可选；若做则小且短时，避免满屏 |
| 状态效果 | 燃烧=橙边，冰冻=青边，感电=白闪 — 在 polygon 描边上做 |

## 资源组织（建议路径）

```
assets/
  shaders/
    glow_sprite.gdshader
    post_process.gdshader
  materials/
    quality_*.tres
  vfx/
    muzzle_flash.tscn
    hit_spark.tscn
```

## 与玩法系统的对齐

- 武器执行序列中的 `play_vfx` 引用 `vfx/` 预设。
- 子弹 `visual` 字段 → `materials/` 中对应 polygon mesh + shader。
- 回合倒计时最后 5s 可轻微提升 vignette 或 UI 脉冲，不遮挡视野。
