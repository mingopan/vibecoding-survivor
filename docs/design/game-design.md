# vibecoding_survivor — 游戏设计总览

## 游戏定位

类幸存者（Survivor-like）Roguelite：单局多回合，回合间在商店构筑武器，核心乐趣来自**武器自定义**——子弹形态、发射方式、命中效果均可组合。

## 单局流程

```
主菜单 → 单局开始
  └─ 回合 1（战斗，限时）→ 商店
  └─ 回合 2（战斗，限时）→ 商店
  └─ …
  └─ 回合 N（Boss / 终局）→ 结算
```

- 每回合怪物**数量**与**类型**由回合配置决定，难度随波次递增。
- 回合时长可配置为 **30s / 60s / 90s**，时间归零即回合结束（击杀清场不作为结束条件）。
- 回合结束获得**代币**，进入商店消费；商店关闭后进入下一回合。

## 设计支柱

1. **构筑深度**：武器扩展（全局 Buff、武器 BD）+ 同型同品质合成，在有限槽位内做取舍。
2. **表达自由**：单把武器的攻击由可编排的**执行序列**驱动，而非固定技能表。
3. **节奏清晰**：固定时长回合 → 商店喘息 → 下一波，形成「紧张—决策」循环。
4. **可读性优先**：极简几何 + 品质色 + 辉光，信息一眼可辨。

## 玩家资源

| 资源 | 获取 | 用途 |
|------|------|------|
| 代币 | 回合结束奖励（可叠加击杀/存活加成） | 商店购买武器、扩展、刷新 |
| 武器槽 | 默认 1，最多 6 | 装备武器本体 |
| 扩展槽 | 每把武器 6~9 个（可配置） | 挂载扩展模块 |

玩家**没有传统背包**：所有战斗相关道具必须进入武器槽或某把武器的扩展槽，否则无法携带进入下一回合。
